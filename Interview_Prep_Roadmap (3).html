Session Flow for Your Friend
Each session follows this 4-phase structure. Your friend leads the interview.
01
First 5 min
Recap Quiz
Friend asks 2–3 questions from the previous session's file. No peeking. Rate answers 1–5 and note confidence in SCHEDULE.md.
02
Next 25–35 min
Interview Q&A
Friend reads each question from the day's file. You answer without looking. After each answer, discuss together — friend uses the "Interview Tip" section to evaluate.
03
Next 10–15 min
Coding on Paper
You explain and code the LeetCode problem on paper or whiteboard — never on an IDE first. Cover: approach, complexity (Time/Space), edge cases.
04
Last 5 min
Rate & Record
Rate confidence (1–5) per question. Note weak spots in SCHEDULE.md tracker. Decide if any topic needs revisit before next session.
Ground Rules
Before Every Session
  • Read the day file's Pre-read section (5 min before)
  • No IDE during session — paper/whiteboard only for code
  • Friend has authority to probe deeper on any answer
  • If confidence < 3 on any topic → add to revision list
  • Don't skip the Recap Quiz — spaced repetition works
⚠ For your friend: The "Interview Tip" section in each question tells you exactly what a good answer looks like and what follow-up questions to ask. Use it as your evaluation rubric — don't just accept a vague answer.
💡 GG9/10 questions should be answered in 60–90 seconds. GG11 questions (design/architecture) deserve 2–3 minutes with examples. Coding problems should be explained first for 2 min before writing any code.
